# Day 1 Exercises

Complete these exercises in Claude.ai as you read. Each one builds a practical skill you'll use daily.

---

## Exercise 1: Elevator Pitch Generator (7 min)

**Scenario:** You need to quickly explain your product or service to different audiences, and each audience cares about different things.

**Steps:**

1. Open a new chat in Claude.ai
2. Paste the following (replace the bullets with your own product, or use these as-is):

```
Here are the key facts about my product:
- Name: TaskFlow Pro
- Category: Project management software for mid-size companies
- Key feature: AI-powered resource allocation
- Result: Reduces project delays by 30% and cuts status meeting time in half
- Price: Starting at $15/user/month

Write a 30-second elevator pitch (about 75 words) for this product.
```

3. Read the pitch Claude generates
4. Now type this follow-up:

```
Great. Now rewrite that pitch for a CFO audience.
Focus on cost savings and ROI instead of features.
Keep it under 75 words.
```

5. Compare the two versions. Notice how Claude shifted the emphasis based on the audience.

**What good looks like:** Two distinct pitches that emphasize different value props — features/productivity for the general pitch, financial impact for the CFO version. Both should be concise and free of jargon.

## Exercise 2: Meeting Recap (8 min)

**Scenario:** You just left a meeting and have messy notes. You need a clean summary and a follow-up email.

**Steps:**

1. Open a new chat in Claude.ai
2. Paste these raw meeting notes (or substitute your own from a recent meeting):

```
Turn these rough meeting notes into a clean summary with action items:

Meeting: Q2 Campaign Planning
Date: Tuesday
Attendees: Sarah (marketing lead), James (content), Priya (demand gen), Mike (sales)

Notes:
- sarah wants to focus on the analytics dashboard launch
- james said he can do 2 blog posts and the landing page copy by end of month
- priya is worried about budget, only 50k for the quarter
- mike wants more case studies, says sales team keeps getting asked for proof points
- agreed to do one webinar in april, priya to handle logistics
- need to finalize messaging by friday
- mike will send over the top 5 objections he hears from prospects
- sarah will set up the project in asana
- next meeting: thursday at 2pm

Format this as:
1. Summary (3-4 sentences)
2. Action items with owner and deadline
3. Key decisions made
```

3. Read the summary
4. Now type:

```
Draft a follow-up email to the attendees. Keep it under 150 words.
Include the action items with owners. Tone: friendly but professional.
```

5. Review the email — it should be ready to send with minimal editing.

**What good looks like:** A well-organized summary that captures the key points, clearly assigned action items with names and deadlines, and a concise follow-up email that's close to send-ready.

## Exercise 3: Job-To-Be-Done Translator (5 min)

**Scenario:** Engineering describes features in technical terms, but your landing page needs customer-facing benefit statements.

**Steps:**

1. Open a new chat in Claude.ai
2. Paste this prompt:

```
You are a conversion copywriter who specializes in B2B SaaS landing pages.

Here are three technical feature descriptions from our engineering team.
Rewrite each one as a customer-facing benefit statement for a landing page.

Feature 1: "Real-time data synchronization across all connected platforms
via REST API with sub-second latency."

Feature 2: "Machine learning model trained on 10M+ project data points
to predict resource bottlenecks 2 weeks in advance."

Feature 3: "Role-based access control with SSO integration supporting
SAML 2.0 and OAuth 2.0 protocols."

For each one:
- Write a benefit-focused headline (under 10 words)
- Write a supporting sentence (under 25 words)
- Use plain language a marketing manager would understand
```

3. Review the three rewrites. They should be jargon-free and focused on what the customer gets, not how it works.

**What good looks like:** Headlines like "Your Data, Always In Sync" instead of "Real-time data synchronization." Supporting copy that speaks to outcomes (save time, avoid problems, stay secure) rather than technical specs.

## Recap

You just practiced three core skills:
- **Providing context** (Exercise 1) — Give Claude the raw material and it produces better output
- **Multi-step conversations** (Exercise 2) — Build on previous responses instead of starting over
- **Role assignment** (Exercise 3) — Tell Claude who to be and it adjusts its writing style
