# Workflow: Team Project Setup Playbook

Use this checklist every time you set up a new shared Project for your team. Copy it into a document or use it directly as a step-by-step guide.

## Step 1: Define the Project

- **Project Name:** Use the format `[Team]-[Initiative]-[Quarter]`
  - Example: `Marketing-ProductLaunch-Q2`
  - Example: `Sales-CompetitiveIntel-2025`
  - For ongoing work, use: `Marketing-BrandVoice-AlwaysOn`
- **Purpose:** Write one sentence describing what this Project is for
- **Participants:** List who should have access

## Step 2: Write Custom Instructions

Include all of these sections in your Project's Custom Instructions:

```
Role: You are a [ROLE] supporting the [TEAM] team at [COMPANY].

Audience: Our target audience is [DETAILED DESCRIPTION — include industry,
company size, job titles, pain points].

Voice: Write in a [TONE] voice. Always [DO THIS]. Never [DON'T DO THIS].

Product/service context:
- [KEY FACT 1]
- [KEY FACT 2]
- [KEY FACT 3]

Constraints:
- All content must comply with [GUIDELINES OR POLICIES]
- Maximum length for [CONTENT TYPE] is [N WORDS]
- Always include [REQUIRED ELEMENTS]
- Never include [PROHIBITED ELEMENTS]

Output defaults: Unless otherwise specified, [FORMAT PREFERENCES —
e.g., use bullet points, keep paragraphs short, include CTAs].
```

## Step 3: Upload Knowledge Files

Upload these types of documents (whatever you have available):

- [ ] Brand or style guide
- [ ] Product documentation or feature sheets
- [ ] Target persona documents
- [ ] Current campaign brief or strategy document
- [ ] Examples of approved content (2-3 pieces that represent your standard)
- [ ] Competitive positioning or battle cards
- [ ] Any relevant compliance or legal guidelines

## Step 4: Test the Project

Before sharing with your team:

1. Start a conversation in the Project
2. Ask for something simple without giving extra context: "Write a LinkedIn post about our latest feature."
3. Check: Does the output reflect your custom instructions? Does it match the tone, audience, and product details?
4. If not, adjust the Custom Instructions and test again
5. Try 2-3 different task types (email, social post, brief) to make sure instructions work broadly

## Step 5: Share with Your Team

1. Add team members to the Project using the sharing settings
2. Send them a brief message explaining:
   - What the Project is for
   - What's in the Custom Instructions (so they understand the context Claude has)
   - What Knowledge Files are uploaded
   - Any conventions to follow (e.g., "start each conversation with the specific task, not background context")
3. Ask for feedback after the first week — refine instructions based on what's working and what isn't

## Ongoing Maintenance

- **Monthly:** Review and update Knowledge Files if product info, messaging, or strategy has changed
- **Quarterly:** Refresh Custom Instructions to match current campaigns and priorities
- **As needed:** Add new Knowledge Files when major documents are created (new case studies, updated positioning, new competitor analysis)
