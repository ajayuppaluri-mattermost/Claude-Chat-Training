# Glossary

Plain-language definitions of terms you'll encounter when using Claude.ai.

---

**Artifact** — A standalone content piece (document, table, code, etc.) that Claude generates in a separate panel next to the conversation. Artifacts can be copied, downloaded, shared, and edited through follow-up messages.

**Attachment** — A file you upload to a conversation using the paperclip icon. Claude can read PDFs, Word documents, CSVs, images, and other file types.

**Claude Code** — A command-line tool for software developers that lets Claude read and write code directly on a developer's computer. Included in Team and Enterprise plans but used by technical teammates, not in the web interface.

**Compliance API** — Enterprise-level tools that let IT administrators monitor and audit Claude usage across the organization. Not something most users interact with directly.

**Context Window** — The total amount of text Claude can "see" in a single conversation, including your messages, Claude's responses, uploaded files, and Project instructions. Team/Enterprise plans have a 200,000-token window — roughly 500 pages.

**Conversation** — A single chat thread with Claude. Each conversation is independent — Claude doesn't remember other conversations unless you're using a Project with shared context.

**Cowork** — A mode where Claude works on longer tasks autonomously in the background, like assigning work to a colleague. You describe the task, Claude works on it, and notifies you when it's ready for review.

**Custom Instructions** — Text you write in a Project that shapes Claude's behavior in every conversation within that Project. Used to define role, audience, voice, constraints, and other persistent context.

**Extended Thinking** — A capability where Claude spends more time reasoning through complex problems before responding. Results in higher quality output for difficult analytical or strategic tasks.

**Hallucination** — When Claude generates information that sounds plausible but is factually incorrect. This is why you should always verify specific claims, statistics, and dates in Claude's output.

**Knowledge File** — A document uploaded to a Project that Claude can reference in every conversation within that Project. Common examples: brand guides, product docs, persona documents.

**Model** — The specific version of Claude you're using. Sonnet is faster for everyday tasks; Opus is the most capable for complex reasoning. You can select the model using the dropdown in the message box.

**MCP (Model Context Protocol)** — A system that lets Claude connect to external tools and services like Google Calendar, Asana, or Slack. Available on Team and Enterprise plans.

**Prompt** — The message you send to Claude. A well-structured prompt includes a role, task, context, and format specification. Better prompts produce dramatically better output.

**Project** — A persistent workspace in Claude.ai with custom instructions and knowledge files. Every conversation within a Project starts with the same context, eliminating the need to re-explain your brand, audience, and goals.

**Role Prompting** — A technique where you tell Claude who to be (e.g., "You are a senior marketing editor") to shape its writing style and perspective. One of the most effective simple techniques.

**Starred Conversation** — A conversation you've pinned for easy access using the star icon. Useful for frequently referenced chats.

**Tokens** — The units Claude uses to measure text. Roughly 1 token = 0.75 words. Token limits determine how much text fits in a conversation or Project.

**Web Search** — Claude's ability to search the internet for current information during a conversation. Enabled by default on Team and Enterprise plans.
